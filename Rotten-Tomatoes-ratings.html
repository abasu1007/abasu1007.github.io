Introduction¶

Rotten Tomatoes gathers movie reviews from critics. An entry on the website typically consists of a short quote, a link to the full review, and a Fresh/Rotten classification which summarizes whether the critic liked/disliked the movie.

When critics give quantitative ratings (say 3/4 stars, Thumbs up, etc.), determining the Fresh/Rotten classification is easy. However, publications like the New York Times don't assign numerical ratings to movies, and thus the Fresh/Rotten classification must be inferred from the text of the review itself.

The task of categorizing text will be performed using Natural Language Processing, in particular by using CountVectorizer and TdIfVectorizer.

Language is incredibly nuanced, and there is an entire field of computer science dedicated to the topic (Natural Language Processing). Nevertheless, we can construct basic language models using fairly straightforward techniques.

For the vectorizer, I tried varying both max_df, min_df, and ngram_range. Increasing ngram_range beyond [1, 2] redices the AUC score, while a min_df of 2 yields the best result. For max_df, I tried increasing to 0.95 which yielded a poorer model.

For the Random Forest model, increasing the n_estimators from 20 to 25 gave better results while further increasing caused the code to take enormous time to run. Increasing the min_samples_leaf from default of 1 to 2 and then to 4 gave progressively better results. Increasing min_samples_split, however, yielded poorer results.

For both CountVectorizer and TfidfVectorizer, binary was set to False.
